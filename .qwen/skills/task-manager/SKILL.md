# Task Manager

Manage development tasks and todos directly in your project using a simple markdown-based task tracking system.

## Quick Commands

```bash
/task-manager add "Implement article search functionality"
/task-manager complete 3
/task-manager list
/task-manager remove 2
```

## Workflow

### Step 1: Task Storage

Tasks are stored in `.ai-factory/TASKS.md` at the project root.

**File format:**
```markdown
# Project Tasks

| ID | Status | Task | Created |
|----|--------|------|---------|
| 1 | ✅ | Setup database migrations | 2026-03-01 |
| 2 | 🔄 | Implement article CRUD | 2026-03-01 |
| 3 | ⏳ | Add search functionality | 2026-03-01 |
```

### Step 2: Add Task

When user asks to add a task:

1. Read `.ai-factory/TASKS.md` (create if doesn't exist)
2. Generate next ID (max ID + 1)
3. Add new row with status `⏳ Pending`
4. Use today's date

**Example:**
```bash
# Input: /task-manager add "Implement user authentication"

# Action: Append to TASKS.md
| 4 | ⏳ | Implement user authentication | 2026-03-01 |
```

### Step 3: List Tasks

When user asks to list tasks:

1. Read `.ai-factory/TASKS.md`
2. Display formatted table
3. Group by status if many tasks

**Output format:**
```
📋 Project Tasks

⏳ Pending:
  [3] Add search functionality
  [4] Implement user authentication

🔄 In Progress:
  [2] Implement article CRUD

✅ Completed:
  [1] Setup database migrations
```

### Step 4: Complete Task

When user marks task complete:

1. Find task by ID
2. Change status from `⏳` or `🔄` to `✅`
3. Optionally add completion date

### Step 5: Remove Task

When user wants to remove:

1. Find task by ID
2. Remove row from table
3. Confirm deletion

## Task Statuses

| Status | Symbol | Meaning |
|--------|--------|---------|
| Pending | ⏳ | Not started |
| In Progress | 🔄 | Currently working |
| Completed | ✅ | Done |
| Blocked | 🚫 | Waiting on something |

## Integration with AI Factory Skills

This skill works alongside the AI Factory workflow:

1. **After `/skills aif-plan`**: Tasks automatically created from plan
2. **During `/skills aif-implement`**: Tasks marked as in-progress/completed
3. **With `/skills aif-commit`**: Reference task ID in commit messages

**Example workflow:**
```bash
/skills aif-plan "Add user registration"
# → Creates tasks in TASKS.md

/task-manager list
# → Shows generated tasks

/task-manager complete 1
# → Mark first task done

/skills aif-commit
# → Commit message: "feat: add user registration form (Task #1)"
```

## Examples

### Add Multiple Tasks
```bash
/task-manager add "Create User entity"
/task-manager add "Add UserRepository interface"
/task-manager add "Implement UserController"
/task-manager add "Write tests for UserController"
```

### Task Progress
```bash
# Start working
/task-manager status 2 in-progress

# Complete
/task-manager complete 2

# View progress
/task-manager list
```

### Bulk Operations
```bash
# Complete all pending tasks
/task-manager complete-all

# Remove completed tasks
/task-manager clean
```

## Edge Cases

1. **Task file doesn't exist**: Create with header and first task
2. **Invalid task ID**: Show error with valid ID range
3. **Duplicate task**: Warn user, suggest similar tasks
4. **Empty task list**: Show helpful message with add command
